# ロト7 予知科学管制システム ― システム概要レポート

> 本レポートは、このアプリ（`app.py` ほか）が「何であるか／どう動くか／何が正直な限界か」を、
> 非エンジニアでも分かるように、かつ後から技術的に追えるようにまとめたものです。
> **秘密情報（APIキー・スプレッドシートURL・個人情報）は一切含みません。** 公開しても安全です。
>
> 最終更新：2026-06-26

---

## 0. ひとことで言うと

**ロト7を「多角的な分析」と「その日の気持ち」で楽しみながら、正直に積み上げていく家族向けの予測・記録アプリ**です。
当選確率そのものは変わりません。狙うのは「**当たったときの取り分（分け前）を大きくする**」「**いろんな可能性を広く張って、どれが近かったかを学ぶ**」こと。
土台にあるのは「夢を買う」という思想で、確率論で夢を否定しない一方、AIは**忖度ゼロ（媚びない）**で弱点や偏りも指摘します。

---

## 1. 全体構成（アーキテクチャ）

| 層 | 使っている技術 | 役割 |
|---|---|---|
| 画面 | **Streamlit**（Python） | Webアプリ本体（`app.py`） |
| 公開・実行 | **Streamlit Community Cloud** | GitHubリポジトリから自動デプロイ |
| ソース管理 | **GitHub**（公開リポジトリ） | コード保管・自動デプロイの起点 |
| データ保存 | **Google スプレッドシート**（gspread） | 当選履歴・予測・記録の保存先（＝DB） |
| AI（主） | **Claude / Anthropic API** | 分析・レポート・占い・反省・監督 |
| AI（第二・任意） | **Google Gemini** | 占い・最終決定の「第二の意見」 |
| 自動運用 | **GitHub Actions（cron）** | 結果取得・採点・収集・週次レポートを無人で実行 |
| 外部データ | 気象庁系（Open-Meteo ERA5）, 各予想サイト | 天気・他サイト予想 |

**デプロイの流れ**：コードを編集 → GitHubにpush → Streamlit Cloudが自動で再起動・反映。
**秘密情報**は Streamlit / GitHub の Secrets と、非公開のGoogleスプレッドシートの中だけにあり、コードには書きません。

### ファイル構成
- `app.py` … アプリ本体（画面・予測エンジン・AI・占い・分析すべて）
- `auto_update.py` … 自動運用スクリプト（headless・GitHub Actionsから実行）
- `.github/workflows/auto-update.yml` … 定期実行スケジュール
- `.streamlit/config.toml` … 画面テーマ（藍×金の宇宙デザイン）
- `requirements.txt` … 依存ライブラリ
- `README.md` … 公開向けの運用ガイド
- `SYSTEM_REPORT.md` … 本レポート

## 2. データ構造（スプレッドシートのタブ）

| タブ | 内容 |
|---|---|
| **実データ** | 当選番号の履歴（第559回〜最新）。六曜・干支・風水・月齢・潮・重力・天気・気温・気圧などの環境列つき |
| **予測ノート** | 日々の積み上げ予測（1口＝1行）。レンズ名・気持ち・キャリーオーバー・採点結果つき。最新が上 |
| **決断記録簿** | 最終決定で確定した買い目とAI講評（追記式・上書きしない） |
| **予想成績ログ** | 各予想サイトの予想を回号別に蓄積。抽選後に採点しランキングの材料に |
| **他サイト予想** | 直近回の他サイト予想（コンセンサス＝共通数字の集計用） |
| **反省ログ** | PDCAの「学び」「総監督講評」を蓄積。次回の予測AIに引き継ぐ |
| **占いラッキー** | 占いが出した今日のラッキーナンバー（日付・利用者別・控えめ反映） |
| **予想サイトURL** | 巡回対象サイトのURL一覧 |

**安全装置**：保存は「全消去→書き直し」方式だが、**空データで全消ししない**ガードを実装済み（通信失敗や誤操作でも記録が丸ごと消えない）。予測ノートは**追記式（上書きしない）**。

## 3. 予測エンジン（心臓部）

### 3-1. 2つの生成モード
- **探索モード（既定）**：30口を**別々の観点（レンズ）に1口ずつ**割り当て、各口にレンズ名を記録。
  抽選後に「どのレンズが近かったか」を学ぶ＝**広く張る／剪定しない**戦略。
- **集中モード**：全レンズを1本の重みベクトルに合成し、超並列シミュレーションで厳選。

### 3-2. レンズ（観点）一覧
過去頻度（土台）／引っ張り・スライド（出目理論）／環境共鳴（暦・干支・九星）／量子シード（その日の気持ちから生成）／AI直感／ドッペルゲンガー（環境が似た過去回）／**人気回避（高数字32〜37＝大穴）**／占いラッキー／**他サイト成績上位（精度重み）**／**他サイト総意（コンセンサス＝共通数字）**／ランダム探索（未知）。

### 3-3. 偏り防止（重要）
過去頻度が強いと特定数字（例：34・12）が全口に張り付く問題があったため、
**「同じ数字は最大8口まで（30口中）」という上限**＋**「まだ少ない数字を優先して穴埋め」**を導入。
→ 検証（60パターン）で「**どの数字も最大8口・1〜37すべてが最低1口は出る**」ことを確認。固定バイアスを機械的に解消。

---

## 4. 最終決定（10億捕捉の包囲網）

積み上げ済みの口から、買う口数（10/20/30）を厳選する工程。

- **スコア**：実績点数（過去頻度の土台）＋気持ちの後押し＋大穴ブースト＋ゆらぎ。
- **大穴（人気回避）狙い**：「大穴を何口入れるか」を数値で指定（既定4）。**ちょうどその数で打ち止め**＝全体が大穴に偏らない（2〜4等も狙えるバランス）。
- **キャリーオーバー連動**：💰チェックで、AIに「分け前最大化」を意識させ、大穴を厚めに。積み上げ時の💰は自動検出。
- **レンズ網羅**：できるだけ多くの観点を1口ずつ取り込み、「沢山の要素を加味」する。
- **偏り防止**：各数字の出現上限・重複（4個以上一致）除外・開始/終了数字の重複制限。
- **AIレポート**：選んだ口（レンズ名つき）とキャリーオーバー状況をClaudeに渡し、忖度ゼロで講評。
- **第二の意見**：ボタンでGeminiが「賛成／反対／見落とし」を別視点で提示。

---

## 5. AI層

- **Claude（主）**：分析・最終決定レポート・PDCA反省・総監督講評・占い。モデルはSecretsで切替（既定 Sonnet、軽い抽出は Haiku、最高品質は Opus）。
- **Gemini（第二・任意）**：占い・最終決定の「第二の意見」。モデル名が変わっても自動でフォールバック。キー未設定でもアプリは壊れない安全設計。
- **プロンプト思想＝忖度ゼロ**：媚びない・気休めを言わない。強みだけでなく弱点・偏り・外れ筋も同じ熱量で指摘。「絶対当たる」式の誇大表現は禁止（確率は動かない事実を踏まえる）。占いも「愛のある厳しさ」。

---

## 6. 他サイト分析（コンセンサス＆精度ランキング）

- **コンセンサス（みんなの共通数字）**：多くのサイトが推す数字を集計し、予測の「他サイト総意」レンズに反映。
- **精度ランキング**：各サイトの予想を回号別に蓄積→抽選後に実データで採点→**サイト別の平均的中でランキング**。当たっているサイトの予想は、積み上げ時に**精度に応じて自動で重め**に反映。
- **データ入力**：自動収集（節約モード＝登録URL＋Haiku抽出／発掘モード＝Web検索）に加え、**確実な手入力フォーム**（サイト名＋7数字）を用意。

---

## 7. 検証と正直さ（忖度ゼロの裏付け）

- **バックテスト**：各レンズが「ランダムより当たるか」を、**先読み防止**（その回より前のデータだけで判定）＋**z値・標準誤差**で検証。多くは「偶然の範囲＝実力なし」と出る前提で、誇張しない。
- **天気×結果の検証**：天気別の出目傾向を集計し、ノイズか本物かをAIが辛口判定。
- **偏り検知／通算成績／レンズ別成績**：自分の予測の偏りを暴き、等級別回数・平均的中・レンズ別累計を可視化（剪定はしない＝学習のため）。

---

## 8. 自動運用（無人で回る部分）

GitHub Actions（cron）で、**毎日の予測と最終決定以外**を自動化：
- **金 05:00 JST**：他サイト予想を収集（節約モード）
- **土 09:00 JST**：結果取得＋全期間更新（天気つき）＋自動採点＋総監督レポート生成（→反省ログへ）

手動は「**気持ちを入れて積み上げる**」「**金曜に最終決定して買う**」の2つだけ。

---

## 9. その他の機能
- **占いの館**：人生・運勢・相性・ラッキーナンバーをAI占い師と対話。公開の相談室＋合言葉で入る非公開の相談室。今日の日付を必ず認識。
- **データを見る**：Sheetsを開かずアプリ内で全タブを閲覧（読み取り専用）。予測ノートは最新回の数字バランス・レンズ内訳・未出数字を自動表示。
- **総監督レポート**：結果→分析→次回を1画面で締める「監督席」。

---

## 10. 正直な限界（誇張しないために）
- **当選確率は1ミリも上がりません。** 本システムの価値は「楽しみ方・記録・学び・分け前狙い・偏り回避」です。
- 他サイトの**自動収集は不安定**（サイト側の事情）。確実なのは手入力。
- レンズの多くは統計的に「実力＝偶然の範囲」と出ます。それでも“広く張る”ことに意味を置く設計です。
- 暦・霊的・重力などの環境要素は、検証上は結果との相関が確認できていません（夢・娯楽として保持）。

## 11. 再利用できる技術要素（他プロジェクト／スキル候補）

このシステムには、ロト7に限らず**他のアプリやスキルに転用できる汎用部品**が含まれています。

1. **Streamlit × Google スプレッドシートDB テンプレート**
   `load_sheet` / `save_sheet`（空データ全消し防止つき）。小規模アプリのDBとして即流用可。
2. **Claude＋Gemini デュアルAIヘルパー**
   `ask_claude` / `ask_gemini`（モデル自動フォールバック・キー未設定でも壊れない）。コスト最適化込み。
3. **GitHub直結デプロイ＋Secrets運用＋GitHub Actions cron**
   「無人で定期処理＋Webアプリ自動デプロイ」の再現可能な型。
4. **多角スコアリング＋偏り防止の選定エンジン**
   重み付き候補から「N個を、偏らず・多様に・クォータ付きで選ぶ」汎用アルゴリズム（推薦・抽選・配分などに応用可）。
5. **先読み防止バックテスト＋有意性検定（z値）**
   「その指標は本当に効くのか」を正直に検証する型。
6. **忖度ゼロAIプロンプト設計**
   媚びない・誇張しない・弱点も指摘するレポート/レビュー用テンプレート。
7. **堅牢スクレイピング**
   文字化け対策（UTF-8固定）＋リトライ＋Web検索フォールバック。
8. **マルチエージェント監査ワークフロー**
   コードを並列で精読→各主張を敵対的に再検証する「正確な調査」の型。
9. **非公開Googleシートの読取（Drive MCP）**
   鍵が手元になくても、接続済みDriveからシート内容を読んで分析する手法。

> これらは [[project-fax-automation]] のような別プロジェクトにも応用できます（特に 1・2・3・6・7）。

## 12. 用語ミニ辞典
- **大穴／人気回避**：みんなが買わない高い数字（32〜37）。当たれば分け前が大きい。
- **キャリーオーバー**：繰越で賞金が大きい週。大穴狙いの価値が上がる。
- **レンズ**：数字を選ぶ観点（過去頻度・引っ張り・占い…）。
- **コンセンサス**：多くの予想サイトが共通して推す数字。
- **忖度ゼロ**：相手に媚びず、弱点や偏りもはっきり言う姿勢。
